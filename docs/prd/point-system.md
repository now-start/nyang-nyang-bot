## 1. 핵심 정의

`호감도`는 치즈냥 채널 커뮤니티의 참여도를 표현하는 잔액형 포인트이다. 출석, 관리자 보상, 업보, 후원 룰렛, 이벤트 참여 결과는 모두 호감도 원장 또는 업보/쿠폰/리워드 보유 목록으로 기록한다.

기본 원칙:

- 사용자의 고유 식별자는 CHZZK `channelId`를 사용한다.
- 닉네임은 표시명이며, 로그인/채팅/후원 이벤트에서 최신 닉네임이 확인되면 갱신한다.
- 히스토리와 원장에는 거래 당시 닉네임 스냅샷을 저장한다.
- 호감도는 숫자 포인트이고, 업보/쿠폰/리워드는 별도 보유 목록으로 관리한다.
- 호감도로 환산 가능한 업보만 교환값을 가진다.
- 호감도로 환산되지 않는 업보는 호감도 잔액에 포함하지 않고 업보 목록에만 표시한다.
- `호감도 +N`처럼 결과가 명확한 포인트 쿠폰은 획득 즉시 자동 전환해 호감도 잔액에 반영한다.
- 시참권, 쿠폰, 방송 리워드처럼 사용처가 있는 항목은 자동 전환하지 않고 업보/쿠폰 목록에 보관한다.
- 호감도 잔액은 음수를 허용하지만, 음수 잔액을 만드는 거래는 관리자만 실행할 수 있다.
- 사용자 임의 사용/교환은 초기 범위에서 제공하지 않고, 사용/교환/룰렛 항목 추가는 관리자가 최종 처리한다.
- 정책 값은 코드에 고정하지 않고 관리자가 조정할 수 있게 한다.
- 방송 재미를 위한 점수이므로 금전성, 환전성, 현금 등가 표현은 금지한다.

## 2. 상태 모델

업보/쿠폰/리워드 상태:

| 상태 | 의미 |
| --- | --- |
| `OWNED` | 사용자가 보유 중 |
| `USED` | 관리자가 사용 처리함 |
| `CONVERTED` | 호감도로 전환됨 |
| `CORRECTED` | 관리자 보정 거래로 정정됨 |

상태 정책:

- 사용자는 상태를 직접 변경할 수 없다.
- `USED`, `CONVERTED`, `CORRECTED` 처리는 관리자 또는 시스템 정책에 의해서만 발생한다.
- 결과 삭제나 취소 상태는 두지 않는다.
- 잘못된 상태 변경은 새 보정 거래와 상태 변경 이력으로 정정한다.

## 3. 관리자 처리 원칙

관리자는 다음 작업을 수행할 수 있다.

- 호감도 지급 또는 차감
- 업보/쿠폰/리워드 지급
- 호감도를 차감해 쿠폰/리워드 구매 처리
- 쿠폰/리워드 구매와 동시에 사용 완료 처리
- 잔액이 부족한 사용자에게 대출성 음수 잔액 생성
- 잘못된 거래를 별도 보정 거래로 정정

정정 원칙:

- 원장 거래는 삭제하지 않는다.
- 잘못된 거래는 원본 거래와 연결된 별도 보정 거래로 정정한다.
- 예: 잘못 지급된 `호감도 +500`은 관리자가 `호감도 지급 -500` 또는 수동 조정 `-500`으로 보정한다.
- 이미 관리자가 사용 처리한 항목을 되돌려야 할 때도 자동 회수가 아니라 관리자 보정 거래로 처리한다.
- 정정 거래는 사용자 히스토리에 공개한다.
- 관리자 내부 메모만 사용자에게 공개하지 않는다.

## 4. 사용자 조회

사용자는 본인 데이터만 조회할 수 있고, 관리자는 전체 사용자 데이터를 조회할 수 있다.

사용자에게 보여야 하는 항목:

- 현재 호감도
- 전체 호감도 변경 히스토리
- 전체 업보/쿠폰/리워드 변경 히스토리
- 누적 업보 반영 합계
- 최근 업보 내역
- 업보 라벨 또는 결과명
- 호감도 변환 점수
- 반영 전/후 호감도
- 반영 출처: `UPBO_MANUAL`, `UPBO_ROULETTE`
- 정정 여부와 원본 거래 연결
- 실행 주체. 예: 시스템, 관리자, 룰렛
- 근거 메모 중 공개 설명

사용자 표시 예시:

| 일시 | 업보 | 호감도 반영 | 반영 후 |
| --- | --- | ---: | ---: |
| 2026-05-07 21:30 | 룰렛: 쓰담 | +10 | 132 |
| 2026-05-07 22:10 | 룰렛: 업보 | -5 | 127 |
| 2026-05-07 22:15 | 정정: 룰렛 업보 보정 | +5 | 132 |

## 5. 원장 모델 후보

현재 `FavoriteHistoryEntity`는 변경 후 호감도와 사유만 저장한다. 최종 포인트 제도에서는 다음 필드가 필요하다.

- `ledgerId`: 변경 이력 ID
- `userId`: 대상 사용자
- `nickNameSnapshot`: 거래 발생 당시 표시 닉네임
- `delta`: 증감값
- `balanceAfter`: 변경 후 잔액
- `reason`: 표시용 사유
- `sourceType`: `ATTENDANCE`, `COMMAND`, `ADMIN`, `UPBO_MANUAL`, `UPBO_ROULETTE`, `SHEET_MIGRATION`, `DONATION`, `SUBSCRIPTION`, `MINI_GAME`, `REWARD_USE`
- `sourceId`: 원본 이벤트 ID
- `displayCategory`: 사용자 화면 분류. 예: `업보`, `출석`, `관리자 보상`
- `publicDescription`: 사용자에게 보여줄 설명
- `privateMemo`: 관리자 전용 메모
- `correctionOfLedgerId`: 정정 거래인 경우 원본 원장 ID
- `actorId`: 관리자 또는 시스템 실행 주체
- `idempotencyKey`: 중복 지급 방지 키
- `createdAt`: 발생 시각
