## 1. 범위

이 문서는 OBS 브라우저 소스에서 표시되는 룰렛 오버레이의 화면 상태, 안전 영역, 애니메이션, 다회차 표시, 오류 표시, 토큰 정책을 정의한다. 관리자 웹 UI의 토큰 발급/재송출 화면은 [웹 UI 명세](web-ui.md)를 따른다.

## 2. 기본 원칙

- 오버레이는 결과를 결정하지 않고 서버 확정 결과를 표시만 한다.
- 대기 상태에서는 완전히 투명해야 한다.
- 방송 화면의 주요 콘텐츠를 과도하게 가리지 않아야 한다.
- 애니메이션 최종 정지 결과는 서버가 전달한 확정 결과와 반드시 일치해야 한다.
- 오류 상태는 방송에 불필요하게 노출하지 않고, 가능한 한 관리자 UI/로그에서 확인한다.
- 토큰 원문은 URL fragment와 `Authorization: Bearer` 헤더로만 사용한다.

## 3. OBS 소스 기준

권장 브라우저 소스:

| 항목 | 기준 |
| --- | --- |
| URL | `/overlay/roulette#token=...` |
| Width | 1920 |
| Height | 1080 |
| FPS | 30 또는 60 |
| Background | Transparent |
| Shutdown source when not visible | 운영 환경에 맞게 선택 |
| Refresh browser when scene becomes active | false 권장 |

지원 해상도:

- 1920x1080 기본.
- 1280x720에서도 정보가 잘리지 않아야 한다.
- 16:9 기준으로 설계하고, 다른 비율은 중앙 안전 영역을 우선한다.

## 4. 안전 영역

- 주요 룰렛 UI는 화면 중앙 70% 폭, 70% 높이 안에 들어와야 한다.
- 하단 자막/채팅 오버레이와 겹치지 않도록 기본 위치는 중앙 또는 중앙 상단을 사용한다.
- 관리자 설정으로 위치 preset을 추후 추가할 수 있다.
  - `CENTER`
  - `TOP_CENTER`
  - `BOTTOM_CENTER`
  - `LEFT`
  - `RIGHT`

MVP에서는 `CENTER`만 구현해도 된다.

## 5. 화면 상태

| 상태 | 설명 | 화면 |
| --- | --- | --- |
| `IDLE` | 표시할 이벤트 없음 | 완전 투명 |
| `CONNECTING` | 토큰 검증 또는 이벤트 조회 중 | 기본은 투명. 관리자 debug mode에서만 표시 |
| `READY` | 이벤트 대기 | 완전 투명 |
| `DISPLAYING` | 룰렛 애니메이션 표시 중 | 룰렛 UI 표시 |
| `SUMMARY` | 초과 회차 요약 표시 | 요약 패널 표시 |
| `DISPLAYED` | 표시 완료 보고 중 | 페이드아웃 후 투명 |
| `ERROR` | 토큰 오류/네트워크 오류 | 기본은 투명, debug mode에서 작은 오류 표시 |

## 6. 룰렛 애니메이션

### 6.1 단회차

흐름:

1. 이벤트 수신.
2. 후원자 닉네임과 룰렛 제목 표시.
3. 항목 리스트 또는 휠이 빠르게 회전.
4. 서버 확정 결과 위치로 감속.
5. 결과 라벨을 강조.
6. 짧은 유지 후 페이드아웃.
7. 표시 완료 API 호출.

권장 시간:

| 구간 | 시간 |
| --- | ---: |
| 진입 | 300ms |
| 회전 | 1800ms |
| 감속 | 900ms |
| 결과 강조 | 1500ms |
| 퇴장 | 300ms |

MVP에서는 총 4~5초 내외를 기본으로 한다.

### 6.2 다회차

규칙:

- 서버는 전체 N회차 결과를 먼저 확정한다.
- 오버레이는 `maxAnimatedRounds`까지 순차 애니메이션으로 표시한다.
- 초과 회차는 요약으로 표시한다.

예:

- 전체 11회
- `maxAnimatedRounds=5`
- 1~5회는 순차 애니메이션
- 6~11회는 결과별 개수 요약

요약 표시 예시:

```text
추가 6회 결과
호감도 +1 x3
꽝 x2
손인사쿠폰 x1
```

## 7. 시각 스타일

오버레이는 [웹 UI 명세의 디자인 시스템 규칙](web-ui.md#21-디자인-시스템-규칙)을 기본으로 공유한다. 다만 OBS 방송 화면의 가독성과 투명 배경 요구사항 때문에 글자 크기, text shadow, 패널 대비는 웹 관리 화면보다 강하게 사용할 수 있다.

### 7.1 톤

- 방송 화면에서 읽기 쉬운 고대비 스타일을 사용한다.
- 배경은 투명이며, 표시 UI는 반투명 패널이나 선명한 텍스트 shadow를 사용한다.
- 너무 큰 장식 요소나 화면 전체를 덮는 효과는 피한다.
- 웹 관리 화면의 배경 그리드, 페이지 패턴, hero 스타일을 오버레이에 가져오지 않는다.

### 7.2 색상

| 용도 | 기준 |
| --- | --- |
| 기본 텍스트 | 밝은 색 |
| 보조 텍스트 | 낮은 대비의 밝은 회색 |
| 긍정 결과 | 청록/초록 계열 |
| 꽝/무효 | 회색 계열 |
| 경고/업보 | 주황/붉은 계열 |
| 희귀 결과 | 금색 또는 밝은 강조색 |

색상만으로 결과 타입을 구분하지 않고 라벨과 아이콘/배지를 함께 사용한다.
색상 토큰은 웹 UI와 같은 청록/주황/붉은 계열을 사용하고, 오버레이 전용 색상은 같은 계열 안에서 명도와 투명도만 조정한다.

### 7.3 타이포그래피

- 1920x1080 기준 결과 라벨은 최소 48px 이상을 권장한다.
- 보조 설명은 최소 28px 이상을 권장한다.
- 긴 라벨은 두 줄까지 허용하고, 세 줄 이상이면 축약한다.
- 폰트 계열은 웹 UI와 맞추되, OBS 환경에서 안정적으로 로드되는 fallback을 둔다.

## 8. 결과 타입별 표시

| resultType | 표시 방식 |
| --- | --- |
| `NONE` | `꽝` 또는 변동 없음으로 표시 |
| `FAVORITE_DELTA` | `호감도 +N`, `호감도 -N` 형식 |
| `UPBO_REWARD` | 업보/리워드 라벨 표시 |
| `COUPON` | 쿠폰 라벨과 보유 안내 |
| `STREAM_MISSION` | 방송 미션 라벨 표시 |
| `PARTICIPATION_PRIORITY` | 시참 우선권 계열 표시 |

오버레이는 결과 안내만 표시하고, 상세 내역은 마이페이지와 `!룰렛결과`에서 확인하도록 한다.

## 9. 토큰과 보안 UX

- URL은 `/overlay/roulette#token=...` 형식을 사용한다.
- fragment token은 초기 HTTP 요청과 access log에 포함되지 않는다.
- htmx 요청은 URL fragment의 토큰을 `Authorization: Bearer` 헤더로 보내며, query string에는 토큰을 싣지 않는다.
- 토큰 오류 시 기본 방송 화면에는 아무것도 표시하지 않는다.
- debug mode가 활성화된 경우에만 작은 상태 문구를 표시한다.
- htmx/Bootstrap 런타임 외 커스텀 외부 스크립트는 사용하지 않는다.

## 10. 이벤트 수신

우선 구현:

- `fetch` 기반 long polling.

대안:

- streaming fetch.
- SSE.
- WebSocket.

MVP 기준:

- 헤더 인증이 가능한 long polling을 우선한다.
- 이벤트가 없으면 투명 상태 유지.
- 동일 이벤트를 중복 표시하지 않는다.
- 이벤트 fragment를 반환할 때 서버가 표시 완료 상태를 기록한다.

## 11. Missed와 Replay

Missed:

- 이벤트가 120초 안에 오버레이로 전달되지 않으면 `MISSED`.
- `MISSED` 이벤트는 자동 재생하지 않는다.
- 방송 화면에는 아무것도 표시하지 않는다.

Replay:

- 관리자가 재송출하면 새 display event를 만든다.
- 재송출 UI는 원본 결과를 그대로 표시한다.
- 재송출 표시에는 원장/보유 목록을 다시 반영하지 않는다.
- 필요하면 관리자용 화면에서만 `재송출` badge를 표시한다. 방송 오버레이에는 기본적으로 표시하지 않는다.

## 12. 오류 처리

| 오류 | 기본 화면 | 운영 확인 |
| --- | --- | --- |
| 토큰 없음 | 투명 | 관리자 OBS URL 확인 |
| 토큰 만료/폐기 | 투명 | overlay token 로그 |
| 네트워크 실패 | 투명, backoff 재시도 | Grafana/Loki |
| 이벤트 payload 오류 | 투명 또는 해당 이벤트 skip | `overlay.failed` 로그 |
| 표시 완료 보고 실패 | 페이드아웃 후 재시도 | display event 상태 |

방송 중 불필요한 오류 문구가 노출되지 않게 한다.

## 13. 접근성과 성능

- 애니메이션은 CSS transform/opacity 중심으로 구현한다.
- layout thrashing이 발생하지 않게 DOM 크기를 안정적으로 유지한다.
- 30fps 환경에서도 결과가 자연스럽게 보여야 한다.
- `prefers-reduced-motion`이 감지되면 회전 시간을 줄이고 결과 강조 중심으로 표시한다.
- 텍스트 shadow 또는 outline으로 가독성을 확보한다.

## 14. 테스트 기준

필수 viewport:

- 1920x1080
- 1280x720

검증:

- 이벤트 없음 상태가 완전 투명이다.
- 단회차 결과가 서버 확정 결과와 일치한다.
- 다회차에서 `maxAnimatedRounds` 이후 결과가 요약된다.
- 120초 만료 이벤트는 자동 표시되지 않는다.
- 재송출은 표시만 수행하고 원장 재반영 API를 호출하지 않는다.
- 토큰 오류가 방송 화면에 노출되지 않는다.
- 긴 라벨이 화면 밖으로 나가지 않는다.

## 15. MVP와 후속 확장

MVP:

- 중앙 배치.
- long polling.
- 기본 단회차/다회차 애니메이션.
- 요약 패널.
- 표시 완료 보고.
- missed/replay.

후속:

- 위치 preset.
- 테마 preset.
- 항목별 사운드.
- 관리자 미리보기 화면.
- 실제 운영 통계 기반 UX 개선.
