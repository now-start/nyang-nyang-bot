## 1. 범위

이 문서는 MVP 구현에 필요한 주요 화면의 저해상도 와이어프레임을 정의한다. 실제 시각 스타일, 색상, 애니메이션 기준은 [웹 UI 명세](web-ui.md)와 [OBS 오버레이 디자인 명세](overlay-design.md)를 따른다.

## 9. 화면 전환 흐름

```text
로그인
  ↓
호감도 보드
  ├─ 내 히스토리 보기 → 마이페이지
  ├─ 관리자: 호감도 수정 모달
  ├─ 관리자: 출석체크 모달
  └─ 관리자: 동기화

관리자
  ├─ 업보 템플릿 관리
  ├─ 업보 수동 적용
  ├─ 룰렛 테이블 관리
  ├─ 오버레이 토큰 관리
  └─ 룰렛 결과 정정/재송출

OBS
  └─ /overlay/roulette#token=...
       ├─ IDLE
       ├─ DISPLAYING
       ├─ SUMMARY
       └─ IDLE
```

## 10. 검증 기준

- 호감도 보드는 360px 모바일 폭에서도 주요 버튼과 텍스트가 겹치지 않는다.
- 관리자 모달은 대상/입력/미리보기/적용 영역이 한 화면 흐름으로 이해된다.
- 마이페이지는 현재 호감도, 보유 항목, 히스토리의 우선순위가 명확하다.
- 룰렛 관리 화면은 확률 합계와 활성화 가능 여부를 즉시 보여준다.
- OBS 오버레이 idle 상태는 완전 투명하다.
- OBS 오버레이 결과 라벨은 1280x720에서도 읽을 수 있다.
