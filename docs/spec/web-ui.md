## 1. 범위

이 문서는 Nyang-Nyang Bot의 웹 화면, 사용자/관리자 UI, 공통 컴포넌트, 상태 표현, 반응형 기준을 정의한다. OBS 브라우저 소스 화면은 [OBS 오버레이 디자인 명세](overlay-design.md)를 따른다.

현재 구현은 `templates/index.html`, `static/css/favorite-list.css`, `static/js/favorite-list.js`, `templates/fragments/*`를 중심으로 호감도 보드, 주간 채팅 순위, 관리자 호감도 조정 모달, 출석체크 모달을 제공한다. 신규 화면은 현행 사용성을 유지하되 클린 아키텍처 adapter 경계에 맞춰 request/response DTO와 화면 상태를 분리한다.

## 2. 디자인 원칙

- 화면은 방송 커뮤니티 운영 도구로서 빠른 조회와 반복 작업을 우선한다.
- 관리자 기능은 장식보다 밀도, 검증, 적용 전 미리보기를 우선한다.
- 일반 사용자는 본인 상태, 최근 변화, 보유 항목을 빠르게 이해할 수 있어야 한다.
- 음수 호감도, 정정 거래, 미확인 항목은 숨기지 않고 명확히 표시한다.
- 금전성, 환전성, 현금 등가 표현은 사용하지 않는다.
- 모든 상태 변경 UI는 성공/실패 피드백을 즉시 제공한다.
- 버튼/입력/테이블/모달은 작은 화면에서도 텍스트가 겹치지 않아야 한다.

## 4. 공통 레이아웃

### 4.1 데스크톱

- 기본 content width는 1080px 이상을 기준으로 하되, 관리자 테이블 화면은 1280px까지 확장 가능하다.
- 상단에는 화면 제목, 현재 사용자/권한, 주요 액션을 배치한다.
- 목록 화면은 검색/필터 영역과 결과 테이블을 분리한다.
- 관리자 화면은 같은 행에서 대상자, 적용값, 결과 미리보기를 볼 수 있어야 한다.

### 4.2 모바일

- 최소 360px 폭에서 주요 텍스트와 버튼이 겹치지 않아야 한다.
- 테이블은 카드형 행 또는 가로 스크롤 중 하나로 명확히 선택한다.
- 관리자 다중 입력 폼은 한 열로 쌓는다.
- 모달은 화면 높이를 넘으면 내부 스크롤을 사용한다.

### 4.3 현행 화면 유지 기준

현행 호감도 보드는 다음 요소를 유지한다.

- hero 영역의 현재 사용자/채널 표시
- 주간 채팅 순위 ticker
- 닉네임 검색
- 호감도 목록과 페이지네이션
- 관리자 전용 출석체크/데이터 동기화/호감도 수정 버튼
- toast, loading, modal fragment

개선 필요:

- 일반 사용자에게 관리자 전용 버튼은 숨기거나 disabled + tooltip 처리한다.
- 히스토리 펼침은 본인 또는 관리자만 가능해야 한다.
- 음수 호감도 행은 색상과 보조 텍스트로 명확히 구분한다.
- 검색 placeholder는 한국어로 통일한다.

## 5. 공통 컴포넌트

### 5.1 버튼

| 종류 | 용도 |
| --- | --- |
| Primary | 저장, 적용, 발급, 재송출 같은 주요 동작 |
| Secondary | 취소, 닫기, 보조 이동 |
| Danger | 폐기, 비활성화, 음수 차감 같은 위험 동작 |
| Ghost | 필터 초기화, 상세 보기, 보조 액션 |
| Icon | 새로고침, 복사, 재발급, 닫기 |

규칙:

- 상태 변경 버튼은 loading/disabled 상태를 제공한다.
- 위험 동작은 확인 모달 또는 명확한 확인 문구를 사용한다.
- 버튼 텍스트는 한 줄에서 잘리지 않게 최소 너비와 줄바꿈 기준을 둔다.

### 5.2 입력

- 필수값은 label과 오류 메시지로 표시한다.
- 숫자 입력은 최소/최대/정수 여부를 즉시 검증한다.
- 호감도 증감값은 `+10`, `-5`처럼 부호를 명확히 표시한다.
- 자유 입력 업보는 라벨, 공개 설명, 관리자 메모를 모두 요구한다.

### 5.3 테이블/목록

- 기본 정렬은 최신순 또는 랭킹순을 명확히 표시한다.
- 히스토리는 최대 50건 페이지 단위로 표시한다.
- 원장/업보/룰렛 결과 목록은 상태 badge를 사용한다.
- 정정 거래는 원본 거래 링크 또는 참조 표시를 제공한다.

### 5.4 모달

- 모달은 제목, 대상, 입력 영역, 미리보기, 액션 영역 순서로 구성한다.
- 적용 전/후 호감도를 항상 표시한다.
- 취소/닫기와 적용 버튼은 시각적으로 구분한다.
- 모달 닫기 시 미저장 입력이 있으면 확인이 필요하다.

### 5.5 토스트와 에러

- 성공 토스트는 짧게 표시한다.
- 실패 토스트는 사용자가 다음 행동을 알 수 있게 작성한다.
- 권한 없음, 네트워크 실패, 검증 실패는 구분한다.
- 장시간 작업은 loading overlay 또는 버튼 내 loading 상태를 표시한다.

## 6. 호감도 보드

목적:

- 인증 사용자가 호감도 랭킹과 주간 채팅 순위를 확인한다.
- 관리자가 사용자를 검색하고 빠르게 수동 조정/히스토리를 확인한다.

필수 요소:

- 현재 사용자 badge
- 주간 채팅 순위
- 닉네임 검색
- 호감도 목록
- 페이지네이션
- 본인/관리자 히스토리 펼침
- 관리자 전용 호감도 수정 버튼

상태:

| 상태 | 표시 |
| --- | --- |
| 목록 없음 | 빈 상태 문구와 검색 초기화 버튼 |
| 검색 결과 없음 | 검색어와 함께 결과 없음 표시 |
| 음수 호감도 | 빨간색 계열 score, `음수` badge 또는 경고 스타일 |
| 히스토리 로딩 | 행 내부 skeleton 또는 `불러오는 중...` |
| 히스토리 권한 없음 | toast + 상세 영역 닫힘 |

## 7. 마이페이지

목적:

- 일반 사용자가 본인 호감도, 업보/쿠폰/리워드, 룰렛 결과, 정정 거래를 확인한다.

필수 섹션:

- 현재 호감도 카드
- 미확인 배지
- 최근 호감도 변경
- 보유 업보/쿠폰/리워드 목록
- 최근 룰렛 결과
- 전체 히스토리

보유 목록:

- 기본 정렬은 `createdAt DESC`.
- 기본 필터는 `OWNED`.
- 상태 필터: `OWNED`, `USED`, `CONVERTED`, `CORRECTED`.
- 환산 불가 항목은 호감도 잔액과 분리해 표시한다.

미확인 배지:

- `lastSeenAt` 이후 신규 보유 항목과 정정 거래 수를 표시한다.
- 페이지 진입 시 읽음 처리한다.
- 정확한 알림 보장이 아니라 best-effort 인지 보조로 취급한다.

## 8. 관리자 호감도/업보 처리

목적:

- 관리자가 호감도 지급/차감, 업보 지급, 쿠폰 구매/사용, 정정 거래를 안전하게 처리한다.

필수 UI:

- 대상 사용자 검색
- 현재 호감도 표시
- 처리 유형 선택
- 템플릿 선택 또는 자유 입력
- 증감값 입력
- 공개 설명 입력
- 관리자 메모 입력
- 적용 전/후 미리보기
- 적용 버튼
- 최근 처리 이력

검증:

- 대상 사용자 필수.
- 자유 입력 시 라벨/공개 설명/관리자 메모 필수.
- 일반 사용자 화면에 관리자 메모가 노출되지 않아야 한다.
- 음수 잔액 생성은 관리자만 가능하며 적용 전 경고한다.
- 정정 거래는 원본 거래 선택 또는 참조 입력이 필요하다.

## 9. 출석체크 UI

목적:

- 관리자가 방송 중 출석체크 사이클을 시작하고 채팅 참여자를 수집해 일괄 지급한다.

필수 UI:

- 사이클 시작 버튼
- 지급 점수 입력
- 수집 대상자 목록
- 대상자 수
- 새로고침/자동 갱신 표시
- 적용 버튼
- 취소 버튼

상태:

| 상태 | 표시 |
| --- | --- |
| 대기 | 시작 버튼 활성 |
| 수집 중 | 대상자 목록과 polling 상태 표시 |
| 적용 중 | 버튼 disabled, loading |
| 적용 완료 | 지급 대상 수와 총 지급량 toast |
| 취소 | 지급 없음 toast |

## 10. 룰렛 관리 UI

목적:

- 관리자가 룰렛 명령어, 1회 금액, 항목, 확률, 결과 타입을 관리한다.

필수 UI:

- 룰렛 테이블 목록
- 활성/비활성 토글
- 명령어 입력
- 1회 금액 입력
- 최대 순차 애니메이션 회차 입력
- 항목 편집 테이블
- 확률 합계 표시
- 꽝 필수 검증 표시
- 1만 회 시뮬레이션 미리보기
- 공개 확률표 미리보기

검증:

- 확률 합계가 100%가 아니면 활성화 불가.
- `꽝`이 없거나 0%이면 활성화 불가.
- 항목이 많으면 OBS/모바일 가독성 경고 표시.
- 소수점 넷째 자리까지 입력 가능.

## 11. 오버레이 토큰 관리 UI

목적:

- 관리자가 OBS 오버레이 토큰을 발급, 재발급, 폐기한다.

필수 UI:

- 토큰 목록
- 이름
- 활성 여부
- 마지막 사용 시각
- 발급자
- 발급 버튼
- 재발급 버튼
- 폐기 버튼
- OBS URL 복사 버튼

보안 UX:

- 토큰 원문은 발급/재발급 직후 한 번만 표시한다.
- 토큰 영역은 자동 숨김 또는 명확한 경고를 제공한다.
- 복사 버튼은 토큰 원문이 표시되는 순간에만 활성화한다.
- 재발급/폐기는 확인 모달을 사용한다.

## 12. 접근성과 반응형

- 모든 입력에는 label이 있어야 한다.
- 모달은 ESC와 닫기 버튼을 지원한다.
- 키보드 포커스 순서는 화면 순서와 일치해야 한다.
- 색상만으로 상태를 구분하지 않고 badge/text를 함께 사용한다.
- 터치 대상은 최소 40px 높이를 권장한다.
- 애니메이션은 기능 이해를 돕는 수준으로 제한한다.

## 13. 테스트 기준

- 호감도 보드는 데스크톱과 모바일에서 텍스트가 겹치지 않는다.
- 일반 사용자는 관리자 버튼을 사용할 수 없다.
- 히스토리 권한 없음은 적절히 표시된다.
- 모달에서 필수값 오류가 inline으로 표시된다.
- 음수 호감도 행은 랭킹과 마이페이지 모두에서 구분된다.
- 오버레이 토큰 원문은 재조회 화면에 표시되지 않는다.
