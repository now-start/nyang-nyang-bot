## 2. 벤치마크 요약

빵떡이는 CHZZK 채팅봇 플랫폼으로, 기본 명령어, 방송 관리, 게임 연동, 음악 시스템, 편의 기능, 자체 재미 기능을 핵심 기능 축으로 제시한다. 명령어 페이지에서는 명령어 등록/삭제, 타이머 명령어, 방송 제목/게임 변경, 쿨타임, 게임 정보 연동, 음악 신청, 가위바위보, 특수 변수 명령어, 카운트/출석체크 등을 확인할 수 있다.

Nyang-Nyang Bot은 우선 출시 범위를 `호감도` 중심 플랫폼으로 제한한다. 빵떡이식 범용 채팅봇 기능은 장기적으로 참고하되 후순위 로드맵으로 확장한다.

## 3. 기능 비교

| 벤치마크 항목 | 빵떡이에서 확인한 방향 | Nyang-Nyang 적용 방향 |
| --- | --- | --- |
| 기본 명령어 관리 | 관리자가 채팅으로 명령어를 등록/삭제하고 권한 등급을 지정한다. | 웹 관리자 화면과 채팅 명령을 모두 지원하되, 명령어 결과에 호감도 변수와 보상 규칙을 연결한다. |
| 타이머/자동 메시지 | 일정 분 단위로 자동 안내 메시지를 실행한다. | 방송 공지, 출석 유도, 호감도 이벤트 안내를 자동 발송한다. |
| 방송 관리 | 방송 제목, 게임 카테고리, 누적 타이머 등 방송 운영 명령을 제공한다. | CHZZK 운영 명령은 후순위로 두고, 호감도 이벤트 운영 도구를 우선 제공한다. |
| 권한 등급 | 일반, 채팅운영자, 채널관리자, 스트리머 등 권한별 명령어 사용을 구분한다. | `USER`, `MANAGER`, `ADMIN`, `STREAMER` 권한 모델로 확장한다. |
| 쿨타임 | 명령어별 기본 쿨타임을 둔다. | 명령어 쿨타임과 포인트 획득 쿨타임을 분리한다. |
| 게임 연동 | LoL, TFT 등 게임 전적/티어 조회 기능을 제공한다. | 초기에는 직접 게임 API 확장보다 호감도 기반 미니게임과 응모 기능을 우선한다. |
| 음악 시스템 | 노래 신청, 취소, 리스트, 현재 곡 조회를 제공한다. | 음악 시스템은 장기 후보로 두고, 호감도 사용처가 분명할 때 신청 우선권/티켓과 연결한다. |
| 미니게임 | 가위바위보 같은 채팅 게임, 순위, 티어, 포인트 요소를 제공한다. | 호감도를 베팅/보상/랭킹에 쓰는 미니게임으로 확장한다. 단, 도박성 설계는 배제한다. |
| 특수 변수 | 닉네임, 업타임, 방제, 시청자 수, 팔로우 기간, 카운트, 출석체크 등 변수를 명령어 템플릿에 삽입한다. | `$호감도`, `$호감도순위`, `$출석연속`, `$이번주채팅순위` 같은 포인트 중심 변수를 제공한다. |
| 카운트/출석 | 명령어 실행 횟수와 사용자별 출석 횟수를 누적한다. | 출석을 단순 횟수가 아니라 호감도 지급 이벤트와 연속 출석 보너스로 모델링한다. |
| 룰렛/오버레이 | 룰렛 확률표, 누적 결과, 오버레이 등 방송 화면용 기능을 제공한다. | 호감도 보상/소모 결과를 방송 오버레이로 노출한다. |
| 공개 링크 | 채널별 명령어 목록, 노래책, 룰렛 확률표 같은 공개 페이지를 제공한다. | 채널 공개 페이지를 만들어 호감도 랭킹, 이벤트 규칙, 보상 내역을 보여준다. |

## 4. 적용 우선순위

1. 호감도 원장과 관리자 수동 조정
2. 업보 템플릿과 수동 적용
3. 명령어 템플릿과 호감도 변수
4. 치지직 후원 룰렛과 OBS 오버레이
5. 공개 페이지와 룰렛 확률표
6. 미니게임/음악/외부 게임 연동
