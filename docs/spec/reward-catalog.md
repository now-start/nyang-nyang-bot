## 1. 범위

이 문서는 호감도 무료 획득 경로, 호감도 사용처, 업보/쿠폰/리워드 교환표, 관리자 수동 입력 정책을 정의한다.

Google Sheets 마이그레이션은 정식 획득 경로가 아니라 기존 포인트의 초기 잔액을 맞추기 위한 전환기 처리이다. 최종 플랫폼에서는 Google Sheets 의존성을 제거한다.

## 2. 무료 획득 경로

### 2.0 출석체크 사이클 모델

- 출석체크는 "활성화 사이클" 단위로 관리한다.
- 한 사이클은 다음 흐름을 가진다.
  1. 관리자가 출석체크를 활성화하면서 부여 점수 `N`(예: 깜짝 출석 +5)을 입력한다.
  2. 활성화 구간 동안 채팅을 친 사용자가 대상자로 수집된다. (시청만 한 사용자는 대상이 아니다.)
  3. 관리자가 `적용`을 누르면 수집된 대상자 모두에게 동일 사이클 내에서 1회 한정으로 호감도 `+N`이 부여된다.
  4. 관리자가 `취소`를 누르면 수집된 대상자에게 호감도가 부여되지 않고 사이클이 종료된다.
- 한 사이클 내에서 동일 사용자에게 복수 부여는 발생하지 않는다.
- 한 방송에 사이클을 여러 개 운영할 수 있다. (예: 첫 출석 +1, 깜짝 출석 +5)
- 사이클 점수 `N`은 정수 양수만 입력하며 상한은 두지 않는다.
- 채팅 기반 자가 신고(예: `!출석`) 방식은 도입하지 않는다.

| 획득 조건 | 지급 호감도 | 판정/처리 방식 |
| --- | ---: | --- |
| 방송 중 불시 출석 호출에 대답 | +1 | 관리자가 출석체크로 대상자를 확인 후 지급 |
| 클립방에 적어놓은 시간이 쇼츠 소스로 사용됨 | +10 | 관리자가 사용 여부 확인 후 지급 |
| 카페에 올린 게시글이 쇼츠 소스로 사용됨 | +5 | 관리자가 사용 여부 확인 후 지급 |
| 컨텐츠 추천이 방송에서 시도됨 | +5 | 치즈냥이 하지 않았던 새로운 컨텐츠만 가능 |
| 컨텐츠 추천 게시판에 올린 영상이 유튜브 영상으로 제작됨 | +5 | 관리자 확인 후 지급 |
| 본인이 추천한 컨텐츠 방송이 유튜브 영상으로 제작됨 | +10 | 관리자 확인 후 지급 |
| 팬아트가 방송/유튜브 소스로 사용됨 | +10 | 관리자 확인 후 지급 |
| 팬영상이 유튜브에 업로드됨 | +30 | 관리자 확인 후 지급 |

정책 메모:

- 일반 채팅 참여는 호감도를 지급하지 않고 주간 채팅 랭킹 집계에만 사용한다.
- 중복 지급 가능 여부는 사유별로 관리자가 판단한다.
- 카페/클립/유튜브 기반 지급은 근거 링크를 남길 수 있어야 한다.
- 주간 랭킹 3위까지는 10% 캐쉬백 대상이다. 기준 금액, 지급 방식, 주간 집계 기준은 별도 운영 정책으로 확정한다.

## 3. 호감도 사용처

초기에는 사용자가 직접 구매/사용하지 않고, 관리자가 포인트 차감, 쿠폰 구매 처리, 즉시 사용 처리를 수동으로 실행한다.

| 사용처 | 필요 호감도 | 조건/비고 |
| --- | ---: | --- |
| 시참 우선권 | 10 | 방송 참여 우선권 |
| 스쿼트 15개 | 150 | 방송 중 리워드 실행 |
| 게임 신청 2-3시간 | 150 | 공포게임 포함 |
| 게임 신청 5-6시간 | 400 | 장시간 게임 신청 |
| 12시간 노방종 | 1000 | 방송 일정/컨디션에 따라 운영자 승인 필요 |
| 즈냥이의 소소한 선물 | 300 | 본인이 해당 달 호감도 1-5등으로 기재된 경우만 구매 가능 |
| 등업 구매 | 100 / 1000 / 5000 | 등업 단계별 필요 호감도는 별도 정의 필요 |

관리자 처리 방식:

- 포인트만 차감하고 사용 완료 처리
- 포인트를 차감해 쿠폰/리워드 구매 처리
- 쿠폰/리워드 구매와 동시에 사용 완료 처리
- 잔액이 부족해도 대출성 처리로 음수 잔액 생성

- 호감도 환산값이 `X`인 업보는 사용자 업보 목록에는 표시하지만 현재 호감도 잔액에는 더하지 않는다.
- 포인트 쿠폰은 사용자 편의를 위해 자동 전환하고, 전환 이력은 호감도 원장에 남긴다.
- 업보를 룰렛 항목으로 추가할 경우, 관리자는 해당 업보의 `일반 교환값 x 5` 만큼 호감도를 차감한다.
- 예: `방송 연장 30분`은 일반 교환값이 호감도 20이므로, 룰렛 항목에 추가하려면 호감도 100을 소모한다.

## 5. 업보 수동 입력

`업보`는 외부 후원사이트의 룰렛후원 결과를 운영자가 확인한 뒤 호감도 또는 쿠폰/리워드로 수동 반영하는 값이다. 초기 제품에서는 외부 후원사이트와 자동 연동하지 않고, 관리자가 방송 중 또는 방송 후 관리 페이지에서 입력한다.

참고 링크: https://cafe.naver.com/f-e/cafes/31192083/articles/3?referrerAllArticles=false

해당 네이버 카페 글은 비회원 접근에서 본문이 제한될 수 있으므로, 업보의 제품 정의는 운영자가 제공한 설명을 기준으로 한다.

관리 페이지 요구사항:

- 업보 결과 템플릿 관리: 라벨, 증감값, 설명, 활성 여부
- 사용자 검색: 닉네임 또는 채널 ID로 대상자 선택
- 업보 수동 입력: 템플릿 선택 또는 자유 입력
- 자유 입력 검증: 라벨, 증감값, 공개 설명, 관리자 메모 필수
- 업보/리워드 사용 처리: 시참, 중참, 쿠폰 사용 등을 수동으로 차감 처리
- 쿠폰 구매 처리: 관리자가 호감도를 차감하고 쿠폰/리워드를 지급
- 구매 즉시 사용 처리: 관리자가 호감도를 차감하고 쿠폰/리워드를 바로 사용 완료로 처리
- 대출성 사용 처리: 잔액이 부족해도 관리자가 승인하면 음수 잔액으로 사용 처리
- 적용 전 미리보기: 현재 호감도, 증감값, 적용 후 호감도
- 적용 이력: 대상자, 증감값, 사유, 실행 관리자, 적용 시각 표시
- 정정: 잘못 입력한 업보를 별도 보정 거래로 정정하고 원장 연결 관계를 남김

거래 유형:

- `업보 사용`
- `시참/중참 사용`
- `호감도 지급`
- `호감도 차감`
- `룰렛 결과 보정`

`호감도 지급` 거래에서도 관리자는 음수 값을 입력해 벌점 또는 대출성 차감을 처리할 수 있다.

## 6. 사용자 인지 정책

- 새로 발생한 보유 업보/쿠폰/리워드와 정정 거래는 본인 마이페이지의 미확인 배지로 노출한다.
- 마이페이지 진입 시 마지막 조회 시각(`lastSeenAt`)을 갱신해 자동으로 읽음 처리한다.
- 시스템은 채팅, 이메일, 외부 푸시 등 별도 채널로 사용자에게 알림을 보내지 않는다.
- 후원자는 본인 룰렛 결과를 다음 경로에서 확인한다.
  - 방송 중 OBS 오버레이 (시청 중일 때)
  - 본인 마이페이지의 `최근 룰렛 결과` 섹션
  - 채팅 명령어 `!룰렛결과` (본인 한정 응답)

## 7. 채팅 명령어 응답 정책

- `!호감도`, `!룰렛결과` 같은 사용자용 채팅 명령어는 사용자별 쿨타임을 적용한다.
- 기본 쿨타임은 `30초`이며 명령어별로 관리자가 변경할 수 있다.
- 같은 사용자가 쿨타임 내에 동일 명령어를 다시 호출하면 시스템은 응답하지 않는다.
- 글로벌(채널 전체) 쿨타임은 별도로 두지 않고, CHZZK 채팅 전송 한도에만 의존한다.
